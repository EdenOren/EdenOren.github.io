# Plan: feature/18-404-page

## Context

Currently the app uses `withHashLocation()` — URLs look like `/#/admin`. The `**` wildcard
route just redirects back to `''` (no real 404 page). Deployed via GitHub Pages.

## Decision: Drop hash routing, use PathLocationStrategy + 404.html copy

GitHub Pages serves `404.html` when a path doesn't exist as a file. By copying `index.html`
to `404.html` in the deploy step, Angular boots on any URL and handles routing client-side.

This gives clean URLs (`/admin` instead of `/#/admin`) with no redirect hacks or sessionStorage
tricks, and lets the real `NotFoundComponent` render for unknown routes.

## What Changes

### 1. Remove `withHashLocation()` — `app.config.ts`
```ts
// before
provideRouter(routes, withHashLocation())
// after
provideRouter(routes)
```
Remove the import too.

### 2. Add `404.html` copy to deploy workflow — `.github/workflows/deploy.yml`
Add one step after Build, before Deploy:
```yaml
- name: Copy index to 404
  run: cp dist/portfolio/browser/index.html dist/portfolio/browser/404.html
```
GitHub Pages will serve `404.html` on any unknown path → Angular boots → router handles it.

### 3. Create `NotFoundComponent` — `src/app/features/not-found/`
Signal Luxe styled page: ghost "404" in display font, short message, CTA back to home.
Matches the section heading pattern (eyebrow → title → subtitle → button).

Files:
- `not-found.component.ts`
- `not-found.component.html`
- `not-found.component.scss`

No facade needed — purely presentational.

### 4. Update wildcard route — `app.routes.ts`
```ts
// before
{ path: '**', redirectTo: '' }
// after
{ path: '**', loadComponent: () => import('./features/not-found/not-found.component').then(m => m.NotFoundComponent) }
```

### 5. Add i18n keys — `src/assets/i18n/en.json`
```json
"NOT_FOUND": {
  "EYEBROW": "Error 404",
  "TITLE": "Page not found",
  "SUBTITLE": "The page you're looking for doesn't exist.",
  "CTA": "Back to home"
}
```
